# Path Engineer 策划案 V0.1

## 0 项目信息
**项目名称**：Path Engineer

**项目类型**：2D模拟建设策略游戏

**目标平台**：Windows

**开发周期**：1 month(Demo)

**技术栈**：C++17 + SFML + JSON

## 1 游戏概述
### 核心理念：
《Path Engineer》是一款融合了路径规划、资源管理和策略决策的模拟建设游戏。玩家扮演道路工程师，需要在复杂地形中规划并建设连接两点的最优路径。

### 游戏特色：
**真实地形模拟**： 多种地形类型，各有不同的建设挑战。

**多样化建设方案**：面对障碍的绕行、架桥、隧道甚至直接移除。

**资源约束管理**：尽可能节约资金预算和工期。

**多目标优化**：平衡成本、时间、环境影响等因素。

## 2 程序模块设计
### 2.1 核心模块（后端）
**GameEngine**：游戏引擎，负责游戏主循环、模块调度和游戏状态管理。

**MapSystem**：地图系统，负责网格地图的生成、地形和障碍物数据的管理。

**ResourceManager**：资源管理，负责预算、时间和劳动力等资源的分配和消耗。

**ConstructionSystem**：建设系统，处理玩家建设指令，计算成本和工期。

**RandomEvent**：随机事件（可选）。
### 2.2 图形与用户界面（前端）
**Render**： 渲染系统，使用SFML绘制2D网格地图、地形、道路、障碍物等。

**TextureManager**：贴图管理器，加载和管理游戏中的贴图资源。

**UIRenderer**：UI渲染，显示资源信息、操作提示和游戏状态。

**InputHandler**：输入处理，处理鼠标和键盘事件，支持建设操作和菜单交互。

### 2.3 关卡系统
**LevelLoader**：从本地文件读取关卡信息。

**LevelSelectionUI**：关卡选择界面。

**LevelEditor**：关卡编辑器（可选）。

## 3 游戏元素设计
### 3.1 地形类型
· **平原**：基础地形，建设成本低。

· **河流**：需要开凿桥梁，成本较高。

· **高山**：需要开凿隧道，成本高且耗时（考虑禁止在隧道拐弯）。

· **沼泽**：需要加固地基，中等成本。

### 3.2 障碍物
· **巨石**：需要清理，增加建设成本和时间。

· **森林**：需要清理，增加建设成本和时间。同时或许可以通过高架桥越过（考虑要不要加设环保这一限制）。

· **湖泊**：大型水域障碍。

· **裂谷/悬崖**：可绕行或架设高架桥（考虑加入施工安全）。

· **遗迹**：不能破坏，必须绕行。

### 3.3 经济系统
· **预算资金**

· **总工期**

· **劳动力个数**：这块有一点小考虑：可以模仿《Civilization》系列，把“工期”这一概念量化为劳动点数，单位时间获得的劳动点数 $=$ 投入劳动力个数 $\times$ 劳动效率。总劳动力个数一定，可以分配在不同地块。（代码难度++++++）

· **工程设备和建筑材料**：这个没必要的话就删掉好了

· **环境指标**：破坏森林？排放超标？河流污染？罚款！（可选）

### 3.4 随机事件（可选）
· **季节、天气系统**：雨雪天气造成的额外开销？大雨冲毁桥梁？山体滑坡砸坏道路？

· **施工安全**：悬崖架桥导致的劳动力减少与额外的赔偿？自然灾害？

· **Bonus**：挖隧道挖出神秘宝藏？
